# 構造編集 仕様書

## 前提

エンジニアがエディタ単体で一から UI を組み立てられること（ケース3）。AI なしで構造の作成・変更が完結する必要がある。

---

## 現状の操作

| 操作 | 手段 | 状態 |
|------|------|------|
| 要素の追加 | コンポーネントパネルからD&D | 実装済み |
| 要素の削除 | Delete キー / 右クリック | 実装済み |
| 要素の移動 | D&D（末尾に追加のみ） | 実装済み（layout-ops-spec.md で改善予定） |
| プロパティ変更 | プロパティパネル | 実装済み |
| Undo / Redo | Ctrl+Z / Ctrl+Shift+Z | 実装済み |

## 不足している操作

一から UI を組み立てるために必要だが、現在できない操作。

---

## 1. コンテナで囲む（Wrap）

### 用途

複数の要素をまとめてレイアウトしたいとき。

```
変更前:
<Screen>
  <Text>タイトル</Text>
  <Button label="送信" />
</Screen>

操作: Text と Button を選択 → VStack で囲む

変更後:
<Screen>
  <VStack>
    <Text>タイトル</Text>
    <Button label="送信" />
  </VStack>
</Screen>
```

### 仕様

- 対象: 選択中の1つまたは複数の要素
- 操作方法: 右クリックメニュー →「囲む」→ コンテナタグを選択（VStack / HStack / Grid / Box / ScrollView / Card）
- キーボードショートカット: なし（メニュー操作のみ）
- 挙動:
  - 選択した要素を元の位置から取り出す
  - 新しいコンテナを元の位置に挿入する
  - 選択した要素をコンテナの子として配置する
  - 複数要素の場合、元の並び順を維持する
  - 操作後、新しいコンテナが選択状態になる

### 複数選択が前提

Wrap は複数要素を囲む操作なので、複数選択の仕組みが必要（後述のセクション6を参照）。単一要素の Wrap も可能（その要素だけをコンテナで囲む）。

---

## 2. 囲みを外す（Unwrap）

### 用途

不要なコンテナを削除して、子要素だけ残したいとき。

```
変更前:
<Screen>
  <VStack>
    <Text>タイトル</Text>
    <Button label="送信" />
  </VStack>
</Screen>

操作: VStack を選択 → Unwrap

変更後:
<Screen>
  <Text>タイトル</Text>
  <Button label="送信" />
</Screen>
```

### 仕様

- 対象: 選択中の1つのコンテナ要素
- 操作方法: 右クリックメニュー →「囲みを外す」
- 挙動:
  - コンテナの子要素を、コンテナの親の中のコンテナがあった位置に挿入する
  - コンテナ自体を削除する
  - 子要素の並び順は維持する
  - 操作後、元の子要素のうち最初の要素が選択状態になる
- リーフ要素（子を持たないタグ）では「囲みを外す」は表示しない
- 子要素がないコンテナでは「囲みを外す」は表示しない（削除を使う）
- ルートコンテナ（親がない要素）では「囲みを外す」は表示しない（挿入先がないため）

---

## 3. 要素の複製（Duplicate）

### 用途

同じ構造の要素をもう1つ作りたいとき。

```
変更前:
<HStack>
  <Card title="商品A" />
</HStack>

操作: Card を選択 → 複製

変更後:
<HStack>
  <Card title="商品A" />
  <Card title="商品A" />   ← 全属性・子要素がコピーされる
</HStack>
```

### 仕様

- 対象: 選択中の1つまたは複数の要素
- 操作方法:
  - 右クリックメニュー →「複製」
  - キーボードショートカット: Ctrl+D
- 挙動（単一選択時）:
  - 選択中の要素とその子孫をディープコピーする
  - コピーした要素を元の要素の直後（兄弟として）に挿入する
  - コピー先の要素には新しい id を割り当てる（全ノードに対して）
  - 操作後、コピーされた新しい要素が選択状態になる
- 挙動（複数選択時）:
  - 選択中の全要素をそれぞれディープコピーする
  - コピーした要素群を、元の最後の選択要素の直後にまとめて挿入する（元の並び順を維持）
  - 操作後、コピーされた新しい要素群が複数選択状態になる

---

## 4. コピー & ペースト

### 用途

離れた位置に要素をコピーしたいとき。Duplicate は直後にしかコピーできないが、Copy & Paste は任意の場所に貼り付けられる。

### 仕様

**単一選択時:**

- コピー: Ctrl+C — 選択中の要素をクリップボードに保存（内部クリップボード）
- カット: Ctrl+X — 選択中の要素をクリップボードに保存し、元の位置から削除する
- ペースト: Ctrl+V — クリップボードの要素を、現在選択中の要素の子の末尾に挿入する
  - 選択中の要素がリーフ要素の場合: 選択中の要素の直後（兄弟として）に挿入する
  - 何も選択していない場合: ルートの末尾に挿入する

**複数選択時:**

- コピー: Ctrl+C — 選択中の全要素をグループとしてクリップボードに保存する。元の並び順を維持する
- カット: Ctrl+X — 選択中の全要素をグループとしてクリップボードに保存し、元の位置から全て削除する
- ペースト: Ctrl+V — クリップボードのグループを、現在選択中の要素の位置に挿入する
  - 単一選択中の場合: その要素の子の末尾に挿入（リーフなら直後に兄弟として挿入）
  - 複数選択中の場合: 最後の選択要素の直後に兄弟として挿入
  - 何も選択していない場合: ルートの末尾に挿入
  - グループ内の要素は元の並び順で連続して挿入される

**共通:**

- クリップボードの内容はエディタ内部で保持する（OS のクリップボードは使わない）
- ペースト時、全ノードに新しい id を割り当てる
- 操作後、ペーストされた要素が選択状態になる（複数の場合は複数選択状態）

## 5. 親の変更（ネストの上げ下げ）

### 用途

要素を現在の親から1つ上の階層に出したいとき。Unwrap は親を消すが、親は残したまま要素だけ外に出す。

```
変更前:
<VStack>
  <HStack>
    <Button label="送信" />
    <Button label="キャンセル" />
  </HStack>
  <Text>注意事項</Text>
</VStack>

操作: 「送信」ボタンを選択 → 親の外に出す

変更後:
<VStack>
  <Button label="送信" />      ← HStack の前に移動
  <HStack>
    <Button label="キャンセル" />
  </HStack>
  <Text>注意事項</Text>
</VStack>
```

### 仕様

- 操作方法: 右クリックメニュー →「親の外に出す」
- 挙動:
  - 選択中の要素を現在の親から取り出す
  - 挿入位置は、要素が親の中で何番目かによって決まる:
    - 親の最初の子 → 親の直前に挿入
    - 親の最後の子 → 親の直後に挿入
    - 親の途中の子 → 親の直後に挿入
  - これによりドキュメント上の前後関係が維持される
- ルート直下の要素では「親の外に出す」は表示しない
- 親が1つの子しか持たない場合も動作する（親は空コンテナとして残る）

## 6. 複数選択

### 用途

Wrap や一括削除など、複数の要素をまとめて操作したいとき。

### 仕様

- 操作方法:
  - Ctrl+クリック: クリックした要素を選択に追加/解除
  - Shift+クリック: 同じ親の兄弟要素を範囲選択
- 選択可能な範囲: **同じ親を持つ兄弟要素のみ**
  - 異なる親の子を同時に選択することはできない
  - 異なる親の子を Ctrl+クリック した場合、前の選択を解除して新しい要素のみ選択する
- 複数選択時のプロパティパネル: 非表示（「{n}個の要素を選択中」と表示）
- 複数選択時に可能な操作:
  - 削除（一括）
  - Wrap（選択した要素をコンテナで囲む）
  - 複製（Ctrl+D）
  - D&D 移動（まとめて移動 — 下記参照）
  - コピー / カット

#### 複数選択 + D&D

複数選択中にいずれかの要素をドラッグすると、選択中の全要素がまとめて移動する。

- ドロップ位置の判定は layout-ops-spec.md の3分割判定と同じ
- 移動先には元の並び順で連続して挿入する
- 自分自身またはその子孫へのドロップは無視する（いずれか1つでも該当すれば全体を無視）
- 複数選択時の表示: 各要素にアウトラインを表示（現行の選択表示と同じ青枠）

## 7. 右クリックメニューの整理

現在の右クリックメニューに構造編集の操作を追加する。

```
現在:
┌──────────────┐
│ スタイルを編集 │
│ ────────────  │
│ 削除          │
└──────────────┘

変更後（単一選択時）:
┌──────────────┐
│ スタイルを編集 │
│ ────────────  │
│ 囲む        ▶ │ → VStack / HStack / Grid / Box / ScrollView / Card
│ 囲みを外す    │ ← コンテナのみ表示
│ 親の外に出す  │ ← ルート直下では非表示
│ ────────────  │
│ 複製          │
│ コピー        │
│ カット        │
│ ペースト      │
│ ────────────  │
│ 削除          │
└──────────────┘

変更後（複数選択時）:
┌──────────────┐
│ 囲む        ▶ │ → VStack / HStack / Grid / Box / ScrollView / Card
│ ────────────  │
│ 複製          │
│ コピー        │
│ カット        │
│ ────────────  │
│ 削除          │
└──────────────┘
```

## 8. ツリービューでの構造編集

左パネルの Tree タブでも、キャンバスと同じ構造編集操作が全て使える。

- 右クリックメニュー: キャンバスと同じメニューを表示（セクション7）
- 複数選択: Ctrl+クリック / Shift+クリック（キャンバスと同じ）
- D&D: layout-ops-spec.md で定義済み（ツリービューでの並び替え）
- キーボードショートカット: 全て有効

ツリービューはネスト構造が明確に見えるため、Wrap / Unwrap / 親の外に出す はキャンバスよりもツリーのほうが直感的に操作しやすい。

## 実装上の留意点

- editorStore に以下の Action タイプが新規に必要:
  - `WRAP_NODES` — 選択ノード群をコンテナで囲む
  - `UNWRAP_NODE` — コンテナを外して子を親に移動
  - `DUPLICATE_NODES` — 選択ノード群を複製
  - `MOVE_OUT` — 要素を親の外に出す
  - `COPY` / `CUT` / `PASTE` — クリップボード操作
- 現在の `selectedNodeId: string | null` を `selectedNodeIds: string[]` に変更する必要がある（複数選択対応）
- 全ての新規操作は Undo/Redo 対応が必須
