# コンポーネント発見 仕様書

## 課題

UIデザイナーではないバックエンドエンジニアがパーツの名称が分からない。

## 前提

エンジニアがコンポーネントを探す場面は主に2つある。

1. **一から UI を組み立てるとき（ケース3）** — どんなコンポーネントがあるか知らない状態から、目的に合うものを探す
2. **AI が生成した .ui を修正するとき（ケース1・2）** — 足りないパーツを追加するために探す

どちらの場合でも、コンポーネントの名前を知らなくても目的のものを見つけられる必要がある。ツールチップのSVGアニメーションで「これが何か」は視覚的に伝わるが、全タグにプレビューがあるわけではなく、説明文も一言で使いどころがわからない。

## 1. 全タグにSVGアニメーションプレビューを追加

### 現状

67タグ中、個別のSVGプレビューがあるのは以下の17タグ。残りはカテゴリ別の汎用プレビューが表示される。

個別プレビューあり（17タグ）:
VStack, HStack, Grid, Button, Input, Toggle, Checkbox, Slider, ProgressBar, Spinner, Avatar, Card, Table, Modal, Chart, Tabs, Toast

個別プレビューなし — スロットタグ以外（41タグ）:
Screen, Box, ScrollView, SplitScreen, HeroSection, BentoGrid, Text, Image, Icon, Divider, Badge, Tag, Skeleton, StatCard, Textarea, Select, Radio, DatePicker, FileUpload, SearchBar, Autocomplete, TagInput, OTPInput, ColorPicker, Link, Navbar, Sidebar, Breadcrumb, Pagination, Stepper, AppBar, Accordion, Carousel, Kanban, Timeline, List, Tree, PricingTable, FAQSection, NotificationPanel, Alert, Tooltip, Popover, ConfirmDialog, EmptyState

スロットタグ（9タグ — 個別プレビュー対象外）:
CardHeader, CardBody, CardFooter, ModalHeader, ModalBody, ModalFooter, TableHeader, TableRow, TableCell

### 仕様

スロットタグ9個を除く全58タグに個別のSVGアニメーションプレビューを追加する（既存17 + 新規41）。各プレビューはそのコンポーネントの特徴的な動作や外観をアニメーションで示す。スロットタグは親コンポーネントの一部として使うため、カテゴリ汎用プレビューのまま据え置く。

SVGのサイズ・スタイルは既存プレビューに合わせる（viewBox: 140x72、色定数: accent, border, bg, text, white）。

## 2. 説明文の充実

### 現状

TAG_DESCRIPTIONS は一言の名称のみ（例: 「スピナー」「統計カード」）。使いどころやどんな場面で使うかがわからない。

### 仕様

各タグの説明文を「何をするか + どんな場面で使うか」の1〜2文に拡張する。

副次効果: 現在の検索フィールドは TAG_DESCRIPTIONS のテキストでもフィルタしている。説明文が充実することで、エンジニアが思いつく言葉でのマッチ率が上がる。例: 「ローディング」で検索 → 改善後は Spinner（「ローディング中の回転アニメーション」）と Skeleton（「コンテンツ読み込み中のプレースホルダー」）がヒットする。
