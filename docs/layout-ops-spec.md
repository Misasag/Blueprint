# レイアウト操作 仕様書

## 概要

キャンバス上での構造変更操作を強化し、「ビジュアルで微調整する」体験を実現する。

---

## 1. D&D による兄弟間の並び替え・任意位置への挿入

### 現状の問題

D&D で要素をドロップすると、常にドロップ先の子の末尾に追加される。兄弟間の順番変更や、任意の位置への挿入ができない。

### 仕様

ドラッグ中に要素の上半分・下半分のどちらにマウスがあるかで挿入位置を判定し、インジケーター（青い線）で挿入位置を表示する。

#### ドロップ位置の判定ロジック

要素の矩形を3分割して判定する。

```
要素の矩形:
┌──────────────────┐
│    上部 25%        │ → この要素の「前」に挿入
├──────────────────┤
│    中央 50%        │ → この要素の「子」として追加（現行動作）
├──────────────────┤
│    下部 25%        │ → この要素の「後」に挿入
└──────────────────┘
```

- 上部 25%: 兄弟として「前」に挿入（同じ親の中で、この要素の直前）
- 中央 50%: 子として追加（現行の動作を維持）
- 下部 25%: 兄弟として「後」に挿入（同じ親の中で、この要素の直後）

※ 子を持てないリーフ要素の場合は、中央 50% も「前」または「後」に振り分ける（上半分 50% → 前、下半分 50% → 後）。

リーフ要素（子を持てないタグ）:
- コンテンツ系: Text, Image, Icon, Divider, Avatar, Badge, Tag, ProgressBar, Skeleton, StatCard, Chart
- 入力系: Button, Input, Textarea, Select, Checkbox, Radio, Toggle, Slider, DatePicker, FileUpload, SearchBar, Autocomplete, TagInput, OTPInput, ColorPicker, Link
- ナビゲーション系: Breadcrumb, Pagination, Stepper
- フィードバック系: Spinner

※ 上記以外のタグ（VStack, HStack, Grid, Box, Card, Modal 等）は子を持てるコンテナ要素。

#### インジケーター表示

ドラッグ中にドロップ位置を視覚的に示す。

```
「前」に挿入の場合:
  ━━━━━━━━━━━━━━━━ ← 青い線（2px solid var(--accent)）を要素の上端に表示
  ┌──────────────┐
  │ <Input>       │
  └──────────────┘

「子」として追加の場合:
  ┌──────────────┐
  │ <VStack>      │ ← 要素全体に薄い青の背景（現行の isDropTarget 動作）
  └──────────────┘

「後」に挿入の場合:
  ┌──────────────┐
  │ <Input>       │
  └──────────────┘
  ━━━━━━━━━━━━━━━━ ← 青い線を要素の下端に表示
```

#### 水平レイアウト（HStack）の場合

親が HStack の場合、判定は上下ではなく左右になる。

```
┌───────┐  ━  ┌───────┐  ━  ┌───────┐
│ 左25% │     │       │     │ 右25% │
│→前に  │     │ 中央  │     │→後に  │
│ 挿入  │     │ 50%   │     │ 挿入  │
└───────┘  ━  └───────┘  ━  └───────┘
```

インジケーターも縦線（要素の左端または右端）で表示する。

#### Grid の場合

Grid は2次元配置のため、上下・左右の両方を考慮する。

```
┌──────────┐
│ 上端 25%  │ → 前に挿入
├──────────┤
│ 中央 50%  │ → 子として追加
├──────────┤
│ 下端 25%  │ → 後に挿入
└──────────┘
```

Grid の子要素はソース順（.ui ファイル上の記述順）で配置されるため、「前」「後」は VStack と同じ上下判定で統一する。左右判定は行わない。

#### 自分自身への D&D 防止

- 自分自身へのドロップは無視する（現行通り）
- 自分の子孫へのドロップも無視する（現行通り）

#### ツリービューでの並び替え

左パネルの Tree タブでも同様の D&D による並び替えを行える。ツリービューはネスト構造が明確に見えるため、キャンバスよりも正確な並び替えに適している。

判定ロジックはキャンバスと同じ（上25% → 前、中央50% → 子、下25% → 後）。インジケーターはツリーノード間に青い線で表示する。

## 2. 子要素のレイアウト属性

### 現状の問題

`grow` と `alignSelf` は LAYOUT セクションに含まれるが、LAYOUT カテゴリ（VStack, HStack 等）のタグにしか表示されない。Button や Input 等の子要素を選択しても、親の中での自分の振る舞いを変更できない。

### 仕様

現在の LAYOUT セクションを2つに分離する。

```
現在の LAYOUT セクション:
  overflow, zIndex, grow, alignSelf → レイアウト系タグのみ表示

変更後:
  LAYOUT（親としてのレイアウト）:
    overflow, zIndex
    → レイアウト系タグ（VStack, HStack, Grid 等）のみ表示

  SELF（親の中での配置）:
    grow, alignSelf
    → 親がレイアウトコンテナ（VStack, HStack, Grid, Box, SplitScreen, BentoGrid）の場合のみ表示
```

`grow` と `alignSelf` は親が flex/grid コンテナでないと効果がない。親がコンテナでない要素（例: Screen 直下の要素）に表示すると、値を変えてもキャンバスが変わらず、「属性を変えたら見た目が変わる」原則に矛盾する。そのため親がレイアウトコンテナの場合のみ表示する。

## 3. 親要素のレイアウト属性への子からのアクセス

### 現状の問題

子要素を選択した状態で、親の `gap`, `align`, `justify` を変更したい場合、一度親をクリックして選択し直す必要がある。

### 仕様

子要素のプロパティパネルに「PARENT LAYOUT」セクションを追加する。親がレイアウトコンテナ（VStack, HStack, Grid, Box, SplitScreen, BentoGrid）の場合のみ表示する。Screen は TAG_PROPS にレイアウト属性（gap, align, justify）がないため対象外。

```
プロパティパネル（子要素 <Button> を選択中）:

┌─────────────────────┐
│ <Button>          ✕ │
│                     │
│ BUTTON              │
│ label   [ログイン  ]│
│                     │
│ SELF                │  ← 新規: 親の中での自分の配置
│ grow    [━━○━ 0   ] │
│ alignSelf [       ] │
│                     │
│ PARENT LAYOUT       │  ← 新規: 親のレイアウト属性
│ ◆ <VStack>          │  ← 親のタグ名を表示
│ gap     [━━○━ 16  ] │
│ align   [          ]│
│ justify [          ]│
│                     │
│ SIZE                │
│ ...                 │
└─────────────────────┘
```

#### 表示ルール

- 親がレイアウトコンテナ（VStack, HStack, Grid, Box, SplitScreen, BentoGrid）の場合のみ表示
- 表示する属性は親の TAG_PROPS のうちレイアウトに関するもの: `gap`, `align`, `justify`, `cols`（Grid の場合）
- 値を変更すると親要素の属性が更新される
- 親のタグ名を表示して、何の属性を変更しているか明確にする
- ルート要素（親がない）の場合は非表示

#### 親 nodeId の取得

PropertyPanel に `parentNode` を渡す。editorStore でノードツリーを走査して、選択中ノードの親を特定する。

```
PropertyPanel に渡す props:
  selectedNode: UINode | null   （現行）
  parentNode: UINode | null     （新規: 選択中ノードの親）

PARENT LAYOUT の属性を変更する場合:
  onUpdateProp(parentNode.id, propName, value) を呼ぶ
```

#### プロパティパネルの表示順

```
CONTENT       （textContent を持つタグのみ）
TAG固有       （TAG_PROPS に定義された属性）
SELF          （grow, alignSelf — 親がレイアウトコンテナの場合のみ）
PARENT LAYOUT （親のレイアウト属性 — 親がレイアウトコンテナの場合のみ）
SIZE
SPACING
APPEARANCE
BORDER
LAYOUT        （overflow, zIndex — レイアウト系タグのみ）
OTHER         （上記に該当しないカスタム属性）
```
